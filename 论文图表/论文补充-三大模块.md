## 5.X 数据看板模块

### 5.X.1 功能描述

数据看板是面向数据分析人员的核心功能模块，提供对 B 站 ACG 视频数据的多维度统计分析与可视化展示。与可视化大屏注重大屏投放和视觉冲击不同，数据看板强调数据深度分析和交互探索能力。本模块包含以下七个子功能：

（1）**概览统计**：展示系统核心数据指标，包括视频总数、用户总数、评论总数、总播放量、总点赞数、总弹幕数、平均播放量共 7 项 KPI 指标卡片。

（2）**30 天数据趋势**：以折线面积图形式展示近 30 天内播放量、点赞数、弹幕数的日级别变化趋势，支持鼠标悬停查看具体数值。

（3）**分类互动率分析**：对各视频分类（动画、游戏、音乐、舞蹈等 12 个分类）进行互动率对比，计算加权互动评分（engagement_score）。评分公式为：
```
互动评分 = 点赞率×0.35 + 弹幕率×0.20 + 评论率×0.20 + 投币率×0.15 + 分享率×0.10
```

（4）**分类平均播放量**：横向柱状图比较各分类的视频数量和平均播放量，辅助分析不同内容类型的受众规模。

（5）**UP 主影响力排行**：按总播放量排序展示 TOP15 创作者，同时显示每位 UP 主的视频数量和总点赞数。

（6）**TOP20 视频数据表**：以表格形式展示播放量最高的 20 个视频，包含标题、播放量、点赞数、弹幕数、点赞率、UP 主等字段，支持点击跳转详情页。

（7）**时长区间分析**：将视频按时长分为 <1 分钟、1-3 分钟、3-10 分钟、10-30 分钟、>30 分钟五个区间，对比各区间视频数量、平均播放量、平均点赞数。

（8）**热门标签云**：统计所有视频标签的出现频率，取 TOP15 以横向柱状图展示。

### 5.X.2 实现逻辑

```
1. 前端加载数据看板页面（hash 路由 #dashboard）
2. 并发调用 7 个仪表板 API 接口：
   - GET /api/dashboard/overview        → KPI 概览卡片
   - GET /api/dashboard/daily-trends    → 30 天趋势图
   - GET /api/dashboard/engagement-analysis → 互动率柱状图
   - GET /api/dashboard/category-stats  → 分类平均播放量
   - GET /api/dashboard/author-ranking  → UP 主排行
   - GET /api/dashboard/top-videos      → TOP20 数据表
   - GET /api/dashboard/duration-analysis → 时长区间分析
   - GET /api/dashboard/tag-cloud       → 标签云
3. 后端通过 SQLAlchemy ORM 执行聚合查询（COUNT、SUM、AVG、GROUP BY）
4. 返回 JSON 格式数据
5. 前端使用 ECharts 5.x 渲染图表：
   - 折线图（smooth + areaStyle 渐变填充）
   - 横向/纵向柱状图（LinearGradient 渐变配色）
   - 双 Y 轴组合图（柱状图 + 折线图）
6. 页面导航切换时调用 dispose() 销毁 ECharts 实例防止内存泄漏
```

    """Per-category engagement rates for deeper analysis"""
    results = db.query(
        Video.category,
        func.count(Video.id).label("count"),
        func.sum(Video.play_count).label("plays"),
        func.sum(Video.like_count).label("likes"),

        func.sum(Video.reply_count).label("replies"),

        func.sum(Video.favorite_count).label("favorites"),
        func.sum(Video.share_count).label("shares"),
    ).filter(Video.category != "").group_by(Video.category).all()

    def safe_rate(numerator, denominator):
        if not denominator or denominator == 0:
            return 0
        return round(numerator / denominator * 100, 2)

    return [
        {
            "category": r[0],
            "video_count": r[1],
            "total_plays": r[2] or 0,
            "like_rate": safe_rate(r[3], r[2]),
            "danmaku_rate": safe_rate(r[4], r[2]),
            "reply_rate": safe_rate(r[5], r[2]),
            "coin_rate": safe_rate(r[6], r[2]),
            "favorite_rate": safe_rate(r[7], r[2]),
            "share_rate": safe_rate(r[8], r[2]),
            "engagement_score": round(
                safe_rate(r[3], r[2]) * 0.35 +   # like_rate weight 35%
                safe_rate(r[4], r[2]) * 0.20 +   # danmaku_rate weight 20%
                safe_rate(r[5], r[2]) * 0.20 +   # reply_rate weight 20%
                safe_rate(r[6], r[2]) * 0.15 +   # coin_rate weight 15%
                safe_rate(r[8], r[2]) * 0.10, 2) # share_rate weight 10%
        }
        for r in results if r[2] and r[2] > 0
    ]

@router.get("/duration-analysis")
def get_duration_analysis(db: Session = Depends(get_db)):
    videos = db.query(
        Video.duration, Video.play_count, Video.like_count, Video.danmaku_count
    ).all()

    buckets = {
        "<1分钟": {"count": 0, "plays": 0, "likes": 0, "danmaku": 0},
        "1-3分钟": {"count": 0, "plays": 0, "likes": 0, "danmaku": 0},
        "3-10分钟": {"count": 0, "plays": 0, "likes": 0, "danmaku": 0},
        "10-30分钟": {"count": 0, "plays": 0, "likes": 0, "danmaku": 0},
        ">30分钟": {"count": 0, "plays": 0, "likes": 0, "danmaku": 0},
    }

    for dur_str, plays, likes, danmaku in videos:
        secs = parse_duration(dur_str)     # "MM:SS" or "HH:MM:SS" → seconds
        if secs < 60:       bucket = "<1分钟"
        elif secs < 180:    bucket = "1-3分钟"
        elif secs < 600:    bucket = "3-10分钟"
        elif secs < 1800:   bucket = "10-30分钟"
        else:               bucket = ">30分钟"
        buckets[bucket]["count"] += 1
        buckets[bucket]["plays"] += (plays or 0)
        buckets[bucket]["likes"] += (likes or 0)
        buckets[bucket]["danmaku"] += (danmaku or 0)

    return [
        {
            "bucket": name,
            "count": d["count"],
            "avg_plays": round(d["plays"] / d["count"]) if d["count"] > 0 else 0,
            "avg_likes": round(d["likes"] / d["count"]) if d["count"] > 0 else 0,
            "avg_danmaku": round(d["danmaku"] / d["count"]) if d["count"] > 0 else 0,
        }
        for name, d in buckets.items() if d["count"] > 0
    ]


def parse_duration(dur_str):
    """Parse 'HH:MM:SS' or 'MM:SS' to seconds"""
    if not dur_str:
        return 0
    parts = dur_str.strip().split(":")
    if len(parts) == 3:
        return int(parts[0]) * 3600 + int(parts[1]) * 60 + int(parts[2])
    elif len(parts) == 2:
        return int(parts[0]) * 60 + int(parts[1])
    return 0
```

## 5.X 可视化大屏模块

### 5.X.1 功能描述

可视化大屏是面向展厅投放、汇报展示等场景的综合性数据展示模块，采用暗色科技主题（深蓝黑背景 + 粉/青/紫渐变配色）。与数据看板的"分析工具"定位不同，可视化大屏强调"视觉冲击力"和"一屏总览"，适合在大型显示器或投影上全屏展示。本模块包含以下八个子功能：

（1）**核心 KPI 指标卡**：展示总播放量、视频总数、总点赞数、平均点赞率 4 项核心指标，以发光数字卡片形式呈现，不同指标使用不同主题色。

（2）**分类分布玫瑰图**：采用 ECharts 南丁格尔玫瑰图（pie + roseType: 'area'），展示各视频分类的数量占比。玫瑰图扇区按角度和半径编码数据，比普通饼图更具视觉层次感。

（3）**热门标签 Top12**：横向柱状图展示出现频率最高的 12 个视频标签，使用线性渐变配色。

（4）**TOP20 视频播放排行**：横向柱状图展示播放量最高的 20 个视频，使用彩虹色渐变条，数据倒序排列使最热视频置于顶部。

（5）**30 天趋势面积图**：三条平滑曲线分别展示播放量、点赞数、弹幕数的 30 天变化趋势，每条曲线下方配有半透明渐变填充，增强数据可读性。

（6）**弹幕活跃度排行**：按分类展示弹幕总数排行，以蓝绿渐变柱状图呈现。弹幕是 B 站 ACG 文化的核心特征，该图表直观反映各内容类型的用户讨论热度。

（7）**UP 主影响力排行**：TOP10 创作者的总播放量排行，使用金橙色渐变配色，突出"流量贡献"的视觉隐喻。

（8）**互动维度雷达图**：六维雷达图（播放、点赞、弹幕、收藏、分享、投币），展示系统整体的互动数据结构。雷达图使用多边形形状、半透明粉色填充。

（9）**全屏模式**：调用浏览器 Fullscreen API 实现一键全屏，适配展厅大屏投放。全屏切换时自动重绘所有 ECharts 实例以适配新尺寸。

（10）**实时时钟**：大屏右上角显示实时日期时间（年-月-日 星期X 时:分:秒），每秒刷新。

### 5.X.2 实现逻辑

```
1. 前端加载大屏页面（hash 路由 #bigscreen）
2. 启动实时时钟定时器（setInterval, 1000ms）
3. 并发调用 8 个仪表板 API 接口获取数据
4. 使用 ECharts 5.x 渲染所有图表：
   - 玫瑰图: series.type='pie', roseType='area'
   - 雷达图: series.type='radar', shape='polygon'
   - 面积图: series.type='line', areaStyle (LinearGradient)
   - 柱状图: series.type='bar', itemStyle (LinearGradient)
5. 监听 fullscreenchange 事件，自动更新按钮文案和图表尺寸
6. 页面离开时清除定时器、销毁所有 ECharts 实例
```

## 5.X 用户收藏模块

### 5.X.1 功能描述

用户收藏模块允许已登录用户收藏感兴趣的 ACG 视频，并在个人收藏页面集中查看所有已收藏视频。本模块包含以下子功能：

（1）**收藏切换**：在视频详情页，登录用户可点击收藏按钮，调用 API 切换收藏状态。已收藏的视频显示"⭐ 已收藏"，未收藏的视频显示"☆ 收藏"。切换机制为幂等操作——同一视频反复点击即在同一接口上完成收藏与取消收藏的切换。

（2）**收藏列表**：在个人收藏页面，以网格卡片形式展示用户已收藏的所有视频，每张卡片包含视频封面、时长标签、标题、UP 主信息。点击卡片可跳转至视频详情页。

（3）**空状态提示**：当用户尚未收藏任何视频时，显示引导提示"还没有收藏"，并提供"去发现视频"按钮跳转至首页。

（4）**收藏状态同步**：视频列表中已收藏的视频在卡片上显示收藏标记；视频详情页根据当前登录用户的收藏状态动态显示按钮样式。

### 5.X.2 实现逻辑

```
收藏切换流程:
1. 前端检查登录状态 → 未登录则弹出"请先登录"提示
2. 已登录 → POST /api/videos/{video_id}/favorite (携带 JWT Token)
3. 后端认证中间件解析 JWT，获取当前用户 user_id
4. 查询 user_favorites 表:
   - 存在记录 → DELETE 删除记录 → 返回 {"favorited": false}
   - 不存在记录 → INSERT 新记录 → 返回 {"favorited": true}
5. 前端根据返回结果更新按钮样式和文案

收藏列表流程:
1. 用户进入收藏页面（hash 路由 #favorites）
2. 前端检查登录状态 → 未登录则跳转首页
3. GET /api/user/favorites?page=1&page_size=100
4. 后端通过 JOIN 查询:
   SELECT * FROM videos
   INNER JOIN user_favorites ON user_favorites.video_id = videos.id
   WHERE user_favorites.user_id = ?
5. 前端渲染视频网格卡片
```

**后端 API — 收藏切换** (`backend/app/routers/videos.py`):

@router.post("/{video_id}/favorite")
def toggle_favorite(
    video_id: int,
    db: Session = Depends(get_db),
    current_user: User = Depends(require_auth),
):
    """Toggle favorite status for a video"""
    video = db.query(Video).filter(Video.id == video_id).first()
    if not video:
        raise HTTPException(status_code=404, detail="视频不存在")

    # 查询是否已收藏
    fav = db.query(UserFavorite).filter(
        UserFavorite.user_id == current_user.id,
        UserFavorite.video_id == video_id,
    ).first()

    if fav:
        # 已收藏 → 取消收藏
        db.delete(fav)
        db.commit()
        return {"favorited": False}
    else:
        # 未收藏 → 添加收藏
        db.add(UserFavorite(user_id=current_user.id, video_id=video_id))
        db.commit()
        return {"favorited": True}
```

**后端 API — 收藏列表** (`backend/app/routers/favorites.py`):

```python
@router.get("/favorites", response_model=VideoListResponse)
def get_favorites(
    page: int = Query(1, ge=1),
    page_size: int = Query(20, ge=1, le=100),
    db: Session = Depends(get_db),
    current_user: User = Depends(require_auth),
):
    """获取当前用户的收藏视频列表"""
    # INNER JOIN 查询用户的收藏视频
    query = db.query(Video).join(
        UserFavorite, UserFavorite.video_id == Video.id
    ).filter(UserFavorite.user_id == current_user.id)

    total = query.count()
    videos = query.offset((page - 1) * page_size).limit(page_size).all()

    items = []
    for v in videos:
        vo = VideoOut.model_validate(v)
        vo.is_favorited = True     # 收藏列表中的所有视频均为已收藏
        items.append(vo)

    return VideoListResponse(
        items=items, total=total, page=page, page_size=page_size
    )
```

## 实现效果

三个模块共用同一套后端 API 服务（FastAPI + SQLAlchemy），通过不同的前端路由和 ECharts 配置实现差异化的展示效果：

| 维度 | 数据看板 | 可视化大屏 | 用户收藏 |
|------|---------|-----------|---------|
| 定位 | 分析工具 | 展示工具 | 个人管理 |
| 图表数量 | 7 个 | 8 个 | — |
| 交互性 | 高（悬停/排序/表格） | 中（全屏切换） | 低（浏览/点击） |
| 配色 | 暗色简约 | 科技炫彩 | 卡片网格 |
| 刷新 | 每次进入重新请求 | 每次进入重新请求 | 每次进入重新请求 |
| ECharts 实例管理 | dispose + resize | dispose + resize + fullscreen | 无 |

**关键设计决策**：数据看板和可视化大屏采用"每次进入页面重新请求全部数据"的策略，而非长连接实时推送。这是因为：
1. B 站视频数据更新频率为天级别，实时推送无必要；
2. 批量请求减少数据库连接开销；
3. 页面切换时销毁 ECharts 实例防止内存泄漏。
