# 4.1 系统用例分析

根据系统的用户体系和功能边界，本系统将使用者划分为两种角色：**访客**（未登录用户）与**普通用户**（已登录用户）。两者具有功能继承关系——普通用户完整继承访客的全部可用功能，并在此基础上扩展仅登录后可用的个人化交互功能。

## 4.1.1 角色定义

| 角色 | 说明 |
|------|------|
| **访客** | 未注册或未登录的系统访问者，无需身份凭据即可使用浏览类功能 |
| **普通用户** | 已注册并登录的用户，持有 JWT 身份令牌，可使用完整的浏览与交互功能 |

## 4.1.2 访客用例

访客角色可使用的功能均为只读浏览类操作，不涉及数据写入或个人状态管理：

**（1）用户注册**
访客通过填写用户名、密码、邮箱和可选昵称完成账号注册。密码经 bcrypt 算法哈希后存储，注册成功后自动签发 JWT 令牌并自动登录。

**（2）用户登录**
已注册用户通过提交用户名和密码完成身份验证。后端验证通过后签发有效期为 7 天的 JWT 令牌，前端存入 localStorage 以供后续请求携带。

**（3）浏览视频列表**
在首页浏览系统收录的视频列表，支持以下多维检索：
- **关键词搜索**：对视频标题、描述、标签、UP 主名称执行全文模糊匹配
- **分类筛选**：按 12 个内容分类（动画、游戏、音乐、舞蹈等）过滤视频
- **多维排序**：按播放量、点赞数、弹幕数或发布时间进行降序/升序排列
- **分页浏览**：每页 20 条视频，支持滑动窗口翻页，始终显示首页与尾页

**（4）查看视频详情**
点击列表中的视频进入详情页，可查看视频的完整信息（标题、封面、UP 主、分类、标签、播放量、点赞数、弹幕数、收藏数、分享数、视频描述等）。

**（5）查看评论**
在视频详情页查看该视频的评论列表，支持顶层评论与子回复的嵌套展示，评论按时间倒序排列。

**（6）浏览数据看板**
访问数据分析模块，查看以下聚合维度的统计图表：
- **概览统计**：系统总体 KPI（视频总数、用户总数、评论总数、总播放量、总点赞数、总弹幕数）
- **分类统计**：按内容分类聚合的视频数量、播放量与点赞量
- **热门视频排行**：按播放量降序排列的 TOP 20 视频
- **30 天趋势**：基于 video_stats_history 表的时序折线图，展示播放/点赞/弹幕三项指标的日级变化
- **分类互动率分析**：按分类计算点赞率、弹幕率、评论率、投币率、分享率及加权互动评分
- **UP 主影响力排行**：按总播放量聚合的 TOP 20 UP 主
- **时长区间分析**：将视频按时长分为五个区间（<1 分钟、1-3 分钟、3-10 分钟、10-30 分钟、>30 分钟），对比各区间视频的平均播放、点赞与弹幕数
- **标签云**：按出现频次统计的 TOP 50 热门标签

**（7）浏览可视化大屏**
访问可视化大屏（深蓝暗色科技主题），以全屏模式呈现系统核心数据的实时可视化展示，包括 KPI 指标卡片、分类分布玫瑰图、TOP 20 视频排行、弹幕活跃度排行、UP 主影响力排行、30 天趋势面积图与互动六维雷达图。大屏支持浏览器原生 Fullscreen API 切换全屏模式，并内置实时数字时钟。

## 4.1.3 普通用户用例

普通用户继承访客的全部用例，并额外获得以下需要登录态的功能（所有接口均通过 `require_auth` 依赖注入强制鉴权）：

**（1）发表评论 / 嵌套回复**
在视频详情页发表顶层评论，或对已有评论进行回复。评论数据通过自引用外键（parent_id）实现嵌套结构，系统限制为一层嵌套（顶层评论 + 子回复），以控制查询复杂度与渲染开销。评论发布成功后即时刷新列表。

**（2）收藏 / 取消收藏视频**
在视频详情页对视频进行收藏操作。系统采用幂等切换设计：同一用户对同一视频反复调用同一接口，即在"已收藏 → 取消收藏"与"未收藏 → 添加收藏"之间切换，前端根据返回的 `favorited` 字段动态更新按钮状态。收藏数据存储在 user_favorites 关联表中。

**（3）个人信息管理**
- **查看个人信息**：查看用户名、昵称、邮箱、头像、注册时间等账户信息
- **修改个人资料**：修改昵称与头像 URL
- **修改密码**：通过输入原密码验证身份后设置新密码（至少 6 位）
- **查看个人统计**：查看本人的评论数、收藏数与注册天数汇总数据

## 4.1.4 用例关系归纳

| 角色 | 用例数 | 功能范围 |
|------|--------|---------|
| 访客 | 7 项 | 注册、登录、浏览视频（搜索/筛选/排序/分页）、查看视频详情、查看评论、浏览数据看板（8 个分析维度）、浏览可视化大屏 |
| 普通用户 | 10 项 | 继承访客全部 7 项 + 发表评论/回复、收藏/取消收藏、个人信息管理 |

普通用户与访客之间的关系为**泛化关系**（一般-特殊关系）：普通用户是访客的扩展角色，访客能够参与的所有用例普通用户均可参与，而普通用户的专有用例对访客不可见（未登录状态下 UI 中的收藏按钮、评论输入框等交互控件自动隐藏或以登录提示替代）。
